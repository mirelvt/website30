Overview of frontend tools and frameworks in the wild.

December 12, 2019

There are lots of frameworks and tools available for web development. And new ones are published while writing this post.

At Dreamsolution the framework we use for our projects is Django. We also have projects created with Flask. But for front end development we use Sass, CSS, Django Templating/HTML and Vanilla JS. In the past we used JQuery for old browser support, but the last few years we don't need to suport IE9 (depends on the clients' customers) so we could skip jQuery and wrote our own js "components" for Ajax, closest, toggle, delegate custom events etc. It is a much smaller js file than using the complete jQuery library. For compiling minified CSS and JS we use GNU Make.

Whether you need framework A or B or no framework, good thing is that they exist and you can learn from it. If you don't need a framework but you know that a specific problem is fixed by one of the frameworks or library, you can check the code to see how they solved it.

I created a list of the frameworks and tools that I've heard/read about. I wanted to make an overview of how much is out there, the list is quite overwhelming. I ordered the frameworks and tools alphabetically.

If you want to know what the most popular tools are at the moment, check out The front end tooling survey 2019, State of CSS 2019, State of Javascript.
